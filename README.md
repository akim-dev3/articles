# Отбор статей по критериям (AI-скрининг)

Веб-инструмент для отбора релевантных статей из большого массива (сотни–десятки тысяч) по критериям, заданным на естественном языке. Анализ названий и аннотаций выполняет нейросеть через [OpenRouter](https://openrouter.ai) — агрегатор LLM с бесплатными моделями, обычно доступный из РФ без VPN (в отличие от Google/OpenAI).

Сайт полностью статический (без сервера/бэкенда) — работает как в браузере на вашем компьютере, так и на GitHub Pages. Все данные (загруженные статьи, результаты, API-ключ) хранятся только локально в вашем браузере (IndexedDB / localStorage) и никуда не передаются, кроме прямых запросов к OpenRouter и, опционально, к сервису Semantic Scholar.

## Как получить бесплатный API-ключ

1. Откройте https://openrouter.ai и зарегистрируйтесь (email, карта не нужна).
2. Перейдите на https://openrouter.ai/settings/keys и создайте ключ (Create Key).
3. Вставьте ключ в поле «API-ключ OpenRouter» на сайте.
4. В поле «Модель» по умолчанию стоит `openrouter/free` — служебный «роутер» OpenRouter, который сам выбирает доступную сейчас бесплатную модель, так что обычно ничего менять не нужно. Если хочется зафиксировать конкретную модель — актуальный список бесплатных: https://openrouter.ai/models?max_price=0, идентификатор вставляется в поле «Модель».

Бесплатные модели OpenRouter ограничены по числу запросов в минуту/день и иногда бывают перегружены — если видите ошибки о превышении лимита, увеличьте паузу между запросами в дополнительных настройках сайта или попробуйте другую модель.

## Как пользоваться

1. **Настройки** — вставьте API-ключ OpenRouter, при желании выберите модель.
2. **Загрузка** — загрузите CSV или Excel-файл со статьями (нужны минимум колонки «название» и «код/DOI/ссылка»). Сопоставьте колонки и подтвердите.
3. **Аннотации (опционально)** — если в файле нет аннотаций, но есть DOI, можно попробовать подтянуть их автоматически через Semantic Scholar. Это повышает точность отбора, но для тысяч статей займёт время. Можно пропустить.
4. **Критерии** — опишите своими словами, что нужно включать, а что исключать, и сколько статей примерно нужно отобрать.
5. **Анализ** — нейросеть обрабатывает статьи пакетами, с прогрессом, паузой и остановкой. Прогресс сохраняется в браузере — можно закрыть вкладку и продолжить позже (уже обработанные статьи не будут пересчитываться заново).
6. **Результаты** — таблица с решениями (включить / под вопросом / исключить), оценкой релевантности и кратким обоснованием. Можно фильтровать, искать и экспортировать в CSV: только включённые, топ-N по релевантности или весь список целиком.

## Локальный запуск

Открыть `index.html` напрямую двойным кликом может не сработать из-за ограничений браузера на модули JavaScript (`file://`). Запустите локальный сервер в папке проекта, например:

```
npx serve .
```

или

```
python -m http.server 8080
```

и откройте `http://localhost:PORT` в браузере.

## Деплой на GitHub Pages

1. Создайте новый репозиторий на GitHub и загрузите в него содержимое этой папки:
   ```
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/<ваш-логин>/<репозиторий>.git
   git push -u origin main
   ```
2. В настройках репозитория откройте **Settings → Pages**, в разделе «Build and deployment» выберите источник **Deploy from a branch**, ветку `main` и папку `/ (root)`.
3. Через пару минут сайт будет доступен по адресу `https://<ваш-логин>.github.io/<репозиторий>/`.

## Ограничения

- Точность отбора зависит от того, что доступно нейросети: только название — грубая оценка; название + аннотация — существенно точнее. Полный текст статей сайт не читает.
- Бесплатные модели OpenRouter ограничивают скорость обработки больших массивов и иногда бывают перегружены (10 000 статей может занять несколько часов, возможны отдельные сбои — их можно повторить кнопкой «Повторить ошибки»).
- Итоговый отбор носит рекомендательный характер — для научной строгости стоит выборочно перепроверить решения нейросети вручную, особенно статьи со статусом «под вопросом».
